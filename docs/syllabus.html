על הקורס

ממה מורכב הקורס ומה תדעו לעשות בסופו.

הקורס מיועד למי שכבר יודע לתכנת ב‑Java ברמת השפה, ורוצה להבין איך תוכנה באמת מגיעה למשתמשים. נלך בדיוק לפי הדרך שקוד עובר במציאות — מכתיבה, דרך בנייה ובדיקות, אריזה ופריסה לענן, ועד ניטור — ונבנה פרויקט אחד שמתפתח שלב‑אחר‑שלב לאורך כל המסלול. אנחנו נמנעים בכוונה מ‑frameworks כבדים — האפליקציה נשארת Java נקי — כדי שכל תשומת הלב תהיה על עולם ה‑Cloud וה‑DevOps.

כל מפגש משלב תיאוריה קצרה, הדגמה חיה, ומעבדה מעשית שבה כל משתתף מתקדם בעצמו על הפרויקט המשותף.

בסיום הקורס תדעו:

  • להבין כיצד תוכנה עוברת מהמחשב המקומי אל הענן (client‑server, HTTP).
  • לבנות צינור CI/CD ב‑GitHub Actions שבונה, בודק ומפרס אוטומטית.
  • לארוז אפליקציה ל‑Container עם Docker ולהריץ אותה בכל סביבה.
  • לעבוד עם שירותי הליבה של AWS בצורה בטוחה ותקציבית (IAM, EC2, S3, מסדי נתונים, Lambda, SQS/SNS).
  • להגדיר תשתית כקוד (Infrastructure as Code) עם Terraform.
  • להוסיף Observability — Logs, Metrics, Traces — ולאבחן תקלות.

קהל יעד ודרישות קדם

למי הקורס מתאים

  • בוגרי כ‑3 חודשי לימוד Java (השפה בלבד) ואלגוריתמים.
  • מי שמכיר משתנים, לולאות, מחלקות, מתודות ואוספים (collections).

מה לא צריך לדעת מראש

לא נדרש ידע קודם ב‑Web, Cloud, Linux, רשתות או DevOps — נבנה את כל הבסיס מאפס במפגש הראשון.

הפרויקט המעשי: Tasks API

חוט שדרה אחד שמלווה את כל 8 המפגשים — נבנה לפי מחזור החיים, שלב‑אחר‑שלב.

נבנה Tasks API — שירות HTTP פשוט לניהול משימות, כתוב ב‑Java נקי על בסיס com.sun.net.httpserver.HttpServer שמובנה ב‑JDK (ללא framework). נקודות הקצה: GET /health, GET /tasks, POST /tasks, GET /tasks/{id}.

נכתוב ונריץ מקומית, נקים CI שבונה ובודק בכל שינוי, נארוז ל‑container, נפרוס לענן, נוסיף נתונים ועיבוד מבוסס‑אירועים (SQS → Lambda → SNS), נגדיר את התשתית כקוד (Terraform), נאוטמט את הפריסה (CD), ולבסוף נוסיף ניטור מלא — בדיוק כמו צוות אמיתי.

ארכיטקטורת הפרויקט — איך הכול מתחבר

תרשים אחד שמראה איך כל נושא בקורס משתלב במערכת אחת חיה.

זרימה מקצה לקצה: מפתח/ת דוחפ/ת קוד ל‑GitHub; GitHub Actions בונה ובודק (CI), ובמיזוג ל‑main מתבצע אימות ל‑AWS עם OIDC, דחיפת image ל‑ECR, terraform apply ופריסה (CD). בענן, ה‑Tasks API רץ כקונטיינר על EC2, שומר משימות ב‑DynamoDB, ובכל יצירת משימה שולח הודעה ל‑SQS; פונקציית Lambda צורכת מהתור ומפרסמת התראה ב‑SNS. IAM שולט בהרשאות, Terraform מקים את כל המשאבים, ו‑CloudWatch + X‑Ray מנטרים את הכול.

1
Code

יסודות והאפליקציה: HTTP, Terminal, Git ו‑Cloud

יסודות

יעדי המפגש

  • להבין את מודל client‑server ופרוטוקול HTTP (בקשה/תגובה, מתודות, קודי סטטוס, פורטים, JSON).
  • להכיר את שורת הפקודה (Terminal) ופקודות Linux בסיסיות.
  • לבצע צעדי Git ראשונים (clone, commit, push) ולקבל את קוד הפרויקט.
  • להבין מהו "ענן" ומהו DevOps, ולהריץ את ה‑Tasks API מקומית.

תוכן המפגש

  • למה DevOps? בעיית "אצלי זה עובד", מסירת קוד מ‑Dev ל‑Ops, ומחזור החיים Code → Observe.
  • HTTP מהיסוד: בקשה/תגובה, GET/POST, קודי סטטוס, פורטים, JSON — הדגמה עם דפדפן ו‑curl.
  • Terminal ו‑Linux: ניווט, קבצים, הרשאות, משתני סביבה, התקנת חבילות.
  • Git בסיסי: clone, commit, push וקבלת ה‑starter repo.
  • מהו הענן: IaaS / PaaS / SaaS, Regions ו‑Availability Zones; סיור בקוד ה‑Tasks API.
מעבדה: clone ל‑starter repo, הידור והרצת ה‑Tasks API מקומית, קריאה ל‑/health ול‑/tasks עם curl ודפדפן, ו‑commit + push ראשון.
משימת בית: התקנת כל כלי הפיתוח; פתיחת חשבון AWS (Free Plan) כבר עכשיו — נשתמש בו ממפגש 4, וההפעלה עשויה לקחת זמן; קריאת דף עזר ל‑HTTP.

Build & Test

אינטגרציה רציפה (CI) עם GitHub Actions

CI/CD

יעדי המפגש

  • להבין מהו CI/CD ולמה לאוטמט כבר מהיום הראשון.
  • לעבוד עם branches, Pull Requests, code review ו‑status checks.
  • להכיר את מבנה GitHub Actions ולבנות pipeline שבונה ובודק בכל push.

תוכן המפגש

  • רענון Git/GitHub: branches, Pull Requests ו‑code review.
  • מהו CI/CD והמודל המנטלי של pipeline; למה לאוטמט.
  • אנטומיה של GitHub Actions: workflows, events, jobs, steps, runners, actions, תחביר YAML; caching ו‑matrix (בקצרה).
  • שלבי ה‑CI שלנו: בניית ה‑JAR (Maven), בדיקות יחידה, lint; secrets, status checks ו‑branch protection על PR.
מעבדה: הוספת .github/workflows/ci.yml — התקנת JDK, בניית הפרויקט עם Maven והרצת בדיקות יחידה; צפייה ב‑pipeline ירוק; הוספת required status check על ה‑branch והוספת בדיקה שנכשלת כדי לראות את ה‑CI תופס.
משימת בית: הוספת בדיקת יחידה נוספת; הוספת build badge ל‑README; קריאה על פירמידת הבדיקות.

Package

קונטיינריזציה עם Docker

DockerCompose

יעדי המפגש

  • להסביר את ההבדל בין Container ל‑VM ולמה קונטיינרים פותרים את "אצלי זה עובד".
  • להבין images מול containers, שכבות (layers) ו‑cache.
  • לכתוב Dockerfile, להריץ container ולדחוף image ל‑registry.
  • להריץ מספר שירותים יחד עם docker‑compose.

תוכן המפגש

  • בעיית האריזה והתאימות; למה קונטיינרים; ארכיטקטורת Docker (daemon, registry).
  • images מול containers, שכבות ו‑build cache.
  • Dockerfile: FROM, COPY, RUN, EXPOSE, CMD/ENTRYPOINT; הרצה עם -p/-e, logs, exec, volumes.
  • docker‑compose: הרצת כמה שירותים בפקודה אחת; דחיפה ל‑Docker Hub; הרחבת ה‑CI ממפגש 2 לבניית ה‑image.
מעבדה: כתיבת Dockerfile ל‑Tasks API; build והרצה עם מיפוי פורטים; בדיקת logs; docker-compose up; דחיפה ל‑Docker Hub.
משימת בית: הקטנת ה‑image (base image רזה) והוספת .dockerignore. (ה‑registry של AWS — ECR — יגיע במפגש 6.)

Deploy · 1

יסודות AWS: IAM, עלויות, מחשוב (EC2 + Lambda)

AWSLambda

יעדי המפגש

  • לנווט בקונסולת AWS בבטחה ולהבין את מודל החיוב החדש (Free Plan, קרדיט של $200 ל‑6 חודשים).
  • להגדיר בקרת עלויות (Budgets והתראות) כדי לא "להישרף" בחיוב.
  • להבין IAM: משתמשים, תפקידים (Roles), הרשאות ועיקרון ההרשאה המינימלית.
  • להבין את ספקטרום המחשוב: EC2 (שרת מנוהל) מול Lambda (serverless), ולפרוס את האפליקציה לענן.

תוכן המפגש

  • סיור בקונסולה + מודל החיוב החדש של AWS; בקרת עלויות: Budgets, התראות, תיוג.
  • IAM: Users / Groups / Roles / Policies, מפתחות מול תפקידים, MFA, הרשאה מינימלית; AWS CLI.
  • EC2: מכונה וירטואלית, AMI, סוגי instance, key pairs, Security Groups.
  • Lambda: מחשוב serverless (FaaS), מודל מונחה‑אירועים, תמחור לפי שימוש; EC2 מול Lambda.
מעבדה: יצירת משתמש IAM + התראת תקציב; הקמת EC2, התחברות ב‑SSH והרצת ה‑Tasks API עם גישה דרך IP ציבורי; פריסת פונקציית Lambda ראשונה.
משימת בית: כיבוי/מחיקת ה‑instance (היגיינת עלויות!), תרגיל קצר ב‑AWS CLI, וקריאה: EC2 מול Lambda.

Deploy · 2

נתונים ואירועים: S3, DynamoDB, SQS/SNS ו‑LocalStack

AWSSQS/SNS

יעדי המפגש

  • להבין אחסון אובייקטים (S3) ומתי משתמשים בו.
  • להבחין בין מסד נתונים רלציוני (RDS) ל‑NoSQL (DynamoDB) ולהכיר מושגי רשת בסיסיים.
  • להבין פירוק תלויות (decoupling) עם תורים והודעות (SQS, SNS) וארכיטקטורה מונחית‑אירועים.
  • להשתמש ב‑LocalStack כדי לתרגל שירותי AWS מקומית וללא עלות.

תוכן המפגש

  • S3: דליים (Buckets), אובייקטים, עמידות (durability), ושימושים נפוצים.
  • מסדי נתונים: RDS מול DynamoDB — מתי כל אחד; מידול בסיסי; רשת ב‑AWS בקצרה (VPC, Subnets, Security Groups).
  • הודעות ופירוק תלויות: SQS (תור) ו‑SNS (פרסום/הרשמה, fan‑out); הארכיטקטורה מונחית‑האירועים: API → SQS → Lambda → SNS.
  • LocalStack: הרצת שירותי AWS מקומית ללא עלות (S3, DynamoDB, SQS, SNS, Lambda).
מעבדה: יצירת S3 bucket; הרצת LocalStack; טבלת DynamoDB ושמירת משימות דרך ה‑AWS SDK for Java; יצירת תור SQS ונושא SNS, חיווט POST /tasks ל‑SQS, ופונקציית Lambda שצורכת מהתור ומפרסמת ל‑SNS — הכול מול LocalStack.
משימת בית: קריאה על ארכיטקטורה מונחית‑אירועים ו‑idempotency.

קונטיינר לענן + תשתית כקוד (ECR + Terraform)

Docker AWSIaC

יעדי המפגש

  • לבנות image בסגנון production עם multi‑stage build ולדחוף ל‑Amazon ECR.
  • להבין תשתית כקוד (Infrastructure as Code) ולמה היא מחליפה הקלקות בקונסולה.
  • להקים משאבי AWS עם Terraform (init / plan / apply / destroy).
  • לפרוס ולהריץ את הקונטיינר על תשתית שהוקמה כקוד.

תוכן המפגש

  • multi‑stage builds (בנייה עם JDK+Maven, ריצה על JRE רזה) — גודל ואבטחה.
  • Amazon ECR: registry פרטי — login, tag, push.
  • Infrastructure as Code: declarative מול imperative, idempotency, state; Terraform: providers, resources, variables; init / plan / apply / destroy (CloudFormation/CDK כחלופות).
  • הרצת הקונטיינר בענן: EC2 + Docker (מסלול חסכוני) מול ECS Fargate (מנוהל) — נפרוס על EC2 שהוקם ב‑Terraform.
מעבדה: multi‑stage build ויצירת ECR repo; כתיבת Terraform שמקים ECR + EC2 + Security Group (terraform apply); דחיפת ה‑image, משיכה והרצה על השרת שהוקם, ואימות; terraform destroy בסוף.
משימת בית: פרמטריזציה של ה‑Terraform עם variables; קריאה על terraform state ו‑remote backends.

Deploy · 4

אספקה רציפה (CD): פריסה אוטומטית ל‑AWS

CI/CD AWSIaC

יעדי המפגש

  • להרחיב את ה‑pipeline ממפגש 2 לבנייה, דחיפה ופריסה אוטומטית.
  • לנהל סודות בבטחה (GitHub OIDC מול IAM Role במקום מפתחות קבועים).
  • להריץ Terraform בתוך הצינור ולהוסיף שער אישור ידני ל‑production.

תוכן המפגש

  • מושגי CD: environments, promotion, approvals, rollback (ברמת מושג).
  • אימות מול AWS מתוך Actions: GitHub OIDC ↔ IAM Role (ללא מפתחות ארוכי‑טווח) מול repo secrets.
  • צינור פריסה מלא: בניית image → דחיפה ל‑ECR → terraform apply בתוך ה‑pipeline → deploy; smoke tests אחרי פריסה.
מעבדה: הרחבת ה‑pipeline — push ל‑main בונה ודוחף image ל‑ECR, מריץ terraform apply/deploy לעדכון הקונטיינר ב‑EC2, ומריץ smoke‑test ל‑/health; הוספת environment עם אישור נדרש.
משימת בית: הפעלת פריסה דרך מיזוג PR; תיעוד ה‑pipeline וה‑IaC ב‑README.

Observe

Observability: Logs, Metrics, Traces + Capstone

Observability

יעדי המפגש

  • להבין את שלושת עמודי ה‑Observability ואת ההבדל מ‑monitoring.
  • לשלוח logs ל‑CloudWatch ולתשאל אותם.
  • ליצור metric, dashboard ו‑alarm.
  • להוסיף trace ולקרוא אותו לאורך השרשרת מונחית‑האירועים (OpenTelemetry → X‑Ray).

תוכן המפגש

  • למה Observability; monitoring מול observability; שלושת העמודים: Logs, Metrics, Traces.
  • Logs: לוגים מובנים (JSON), CloudWatch Logs, log groups/streams, ותשאול עם Logs Insights.
  • Metrics: CloudWatch metrics, dashboards ו‑alarms; metric מותאם; רעיונות RED/USE (בקצרה).
  • Traces: tracing מבוזר, spans, OpenTelemetry auto‑instrumentation (-javaagent) → AWS X‑Ray; קריאת trace לאורך API → SQS → Lambda.
מעבדה + Capstone: שליחת logs ל‑CloudWatch; dashboard + alarm על שגיאות/latency; חיבור OTel Java agent וצפייה ב‑trace דרך API → SQS → Lambda ב‑X‑Ray. סיום בריצה מלאה: push → CI/CD מפרס → תעבורה → logs, metric+alarm ו‑trace.
סיכום: צ׳קליסט ניקוי עלויות (terraform destroy וכו׳); לאן ממשיכים מכאן.

הערכה והשתתפות

ההערכה מבוססת על התקדמות מעשית, לא על מבחנים:

  • השתתפות במעבדות — השלמת המעבדה בכל מפגש.
  • התקדמות הפרויקט — כל שלב בקשת ההתפתחות נבנה על קודמו.
  • הדגמת Capstone — הצגת השירות רץ בענן עם CI/CD וניטור פעיל במפגש האחרון.
